# User Stories (Day 1 Frozen Set)

1. As a support agent, I want to ask whether an unopened item can be refunded within 14 days and receive a cited answer.
2. As a support agent, I want to retrieve the latest order status instead of trusting model memory.
3. As a support agent, I want to retrieve customer profile context by identifier before proposing an action.
4. As a support agent, I want the assistant to draft a refund proposal instead of directly issuing a refund.
5. As a support agent, I want an address-change request to require approval when shipment is already in transit.
6. As a support agent, I want cancellation requests to be converted into approval-gated proposals.
7. As a support lead, I want risky requests to be escalated to human review.
8. As a support lead, I want a short structured summary attached to each escalated ticket.
9. As a support lead, I want proposed actions to include an approval ID and approval status.
10. As a developer, I want to inspect tool traces when a case fails.
11. As a developer, I want deterministic tool JSON contracts so tests catch contract drift.
12. As a developer, I want a fixed eval set covering routing, retrieval, risky actions, and approval paths.
13. As a hiring reviewer, I want three stable golden demo flows that show policy QA, order lookup, and a sensitive action workflow.
